## 1. 项目概述

OpenBiliClaw 是一个**纯本地、私有、开源的自进化跨平台内容发现 Agent**。它从你的跨平台使用行为、反馈和对话中持续深化心理画像，带着对你的理解主动去 B 站、小红书、抖音、YouTube、X（Twitter）等来源寻找你会喜欢的内容，并用**朋友式的语言**告诉你为什么推荐它。

名字起源于"BiliClaw"（B 站的小爪子），从 v0.3.0 起已扩展为通用跨平台 Agent。

### 一句话理解

> **不是"猜你喜欢"的推荐算法，而是一个住在你电脑里的 AI 朋友——先深度理解你这个人，再跨平台主动帮你找好东西，并且会像朋友一样告诉你"为什么我觉得你会喜欢这个"。**

## 2. 为什么需要 OpenBiliClaw

### 2.1 传统推荐系统的问题

推荐系统本质上是一个**中间商**。它站在海量内容和海量用户之间做匹配分发。现代推荐系统同时权衡点击率、完播率、点赞概率、停留时长、用户留存、创作者生态、广告收入等十几个目标，把它们加权压成一个分数来排序。

**核心矛盾：这些权重是平台定的，优化目标归根结底是平台的。** 用户满意度只是被当作留存和变现的手段，而非目的本身。

| 问题 | 说明 |
|------|------|
| **信息茧房** | 推荐越来越像你已经看过的东西，偶尔的惊喜全靠运气 |
| **平台孤岛** | 你在 B 站看了三年机械键盘，小红书完全不知道；你的兴趣被割裂在不同平台的数据库里 |
| **不可解释** | "猜你喜欢"——但从来不说为什么猜、依据是什么 |
| **无法调教** | 只能点"不感兴趣"，无法告诉系统你真正想要什么 |
| **数据不归你** | 你的行为数据是平台的资产，不是你自己的 |

### 2.2 OpenBiliClaw 的做法

| | 各平台官方推荐 | 关键词过滤插件 | **OpenBiliClaw** |
|---|---|---|---|
| 推荐逻辑 | 协同过滤 | 标签匹配 | **心理画像 + 五层记忆** |
| 内容来源 | 单一平台 | 单一平台 | **跨平台（B 站 · 小红书 · 抖音 · YouTube · X）** |
| 信息茧房 | 越推越窄 | 不解决 | **猜测兴趣主动破茧** |
| 数据归属 | 平台所有 | 通常云端 | **100% 本地 SQLite** |
| 推荐解释 | "猜你喜欢" | 无 | **像朋友一样告诉你为什么** |
| 可定制 | 不可以 | 低 | **换 LLM / 改画像 / 写 Skill** |

## 3. 核心原理：推荐是怎么工作的

OpenBiliClaw 的推荐与其他系统最大的不同在于：**它是从"你是什么样的人"出发，而不是从"视频有什么标签"出发。**

整条推荐链路分为四个阶段：

```
理解你 → 找内容 → 评估内容 → 推荐给你
  (灵魂引擎)   (发现引擎)   (评估引擎)   (推荐引擎)
```

### 3.1 第一阶段：理解你——五层灵魂引擎

这是整个系统最核心、最具差异化的部分。系统不只记录"你看了什么"，而是通过五层递进的理解模型，从行为数据中构建对你的深度认知：

```
🌟 灵魂层 (Soul Layer) —— "他是一个什么样的人"
    ↑↓ 双向修正
💡 洞察层 (Insight Layer) —— "为什么他会这样"
    ↑↓ 双向修正
📅 觉察层 (Awareness Layer) —— "每天他在发生什么变化"
    ↑↓ 双向修正
📊 偏好层 (Preference Layer) —— "他喜欢什么/不喜欢什么"
    ↑↓ 数据提取
📝 事件层 (Event Layer) —— "他做了什么"
```

**每一层的具体内容：**

| 层 | 包含内容 | 示例 |
|----|---------|------|
| **事件层** | 点击、观看、搜索、收藏、点赞、评论、停留时长、滚动行为 | "今天看了 3 个机械键盘评测视频，在其中一个停留了 25 分钟" |
| **偏好层** | 兴趣标签（带权重+时间衰减）、风格偏好、情境模式、不喜欢的话题 | "机械键盘(权重0.85) · 数码评测(0.72) · 不喜欢：营销号解说" |
| **觉察层** | 每日观察笔记、兴趣趋势变化、情绪状态推测 | "最近三周摄影相关观看增加了 40%，可能是新兴趣在萌芽" |
| **洞察层** | 动机分析、心理需求推断、潜在兴趣假设、行为模式归因 | "他对机械键盘的兴趣可能不只是实用需求——他关注的 UP 主都强调设计和手感，这意味着他可能在审美层面有更强的驱动力" |
| **灵魂层** | MBTI 人格类型、核心价值观、深层需求、认知风格、生活状态 | "INTP 型，核心驱动是'理解世界运作方式'，偏好深度分析而非浅层娱乐" |

**关键设计：层间是网状而非单向的。** 上层理解会指导下层数据的解读（比如灵魂层判断你是 INTP 型后，偏好层会更关注分析类内容），下层新数据也会修正上层理解（比如你突然看了很多游戏实况，洞察层需要解释这个变化）。

### 3.2 第二阶段：找内容——多策略发现引擎

系统不会被动地等你刷推荐流，而是**主动出击**，通过多种策略寻找内容：

| 策略 | 说明 | 示例 |
|------|------|------|
| **兴趣关键词搜索** | 根据画像生成关键词组合搜索 | 你的画像是"喜欢深度技术分析"，生成"机械键盘 结构原理 长评测"去搜索 |
| **相关推荐链探索** | 从已知好内容出发，沿相关推荐不断深入 | 从这个视频的相关推荐里发现另一个好 UP 主 |
| **分区热门/排行榜** | 扫描各分区热门，结合你的画像筛选 | 科技区热门 100 个视频里，只有 3 个符合你的口味 |
| **UP 主追踪** | 追踪你关注的和系统发现的优质 UP 主 | 你关注的 UP 主更新了，或者发现了一个风格类似的新 UP 主 |
| **跨领域探索** | **刻意推荐你从未接触过但心理画像暗示可能喜欢的领域** | 你喜欢机械键盘的工艺美学 → 也许你会喜欢建筑设计的结构美学 |
| **热点关联** | 追踪热点话题，判断是否与你的深层兴趣相关 | 苹果发布会热点 + 你关注科技原理 → 推荐深度技术解读而非新闻快讯 |

**跨领域探索是 OpenBiliClaw 最独特的能力之一。** 它基于心理学桥接逻辑：一个关注机械表的人可能会喜欢建筑美学，一个看量子物理科普的人可能对哲学感兴趣。协同过滤永远不会推给你"没人从这条路径走过"的内容，但 OpenBiliClaw 会。

### 3.3 第三阶段：评估内容——以你为准的评分

所有发现的内容进入**统一待评估池**，由 LLM 结合你的完整灵魂画像进行评分。评分维度包括：

- **相关性**：这个内容是否匹配你的深层兴趣和当前状态？
- **新鲜度**：你是否最近看过类似的内容？
- **来源多样性**：是否来自不同的平台和创作者？
- **惊喜度**：这个内容是否在你舒适区之外但很可能喜欢？
- **话题疲劳度**：同一话题是否已经推过太多次？

### 3.4 第四阶段：推荐呈现——像朋友一样告诉你为什么

这是 OpenBiliClaw 与所有推荐系统最直观的区别。它不是给你一个冷冰冰的列表，而是**像朋友一样解释推荐理由**。

**传统推荐系统：**
> "因为您观看了相关视频，为您推荐以下内容" ❌

**OpenBiliClaw：**
> "我觉得你会喜欢这个——这个 UP 主讲 AI 的角度很独特，有点像你喜欢的那种'把复杂的事情讲透'的风格。我理解你最近对 AI 的关注不仅是工作需要，更多是一种对未来的好奇，这个视频正好聊到了你可能感兴趣的方向。" ✅

## 4. 图文推荐与推荐理由生成

### 4.1 推荐卡片包含的信息

每条推荐都是一张**图文卡片**，包含：

| 信息 | 说明 |
|------|------|
| **封面图/文字卡** | 视频封面或推文/笔记的纯文字卡渲染（通过图片代理安全加载） |
| **标题** | 内容标题 |
| **来源平台** | B 站 / 小红书 / 抖音 / YouTube / X |
| **UP 主/作者** | 内容创作者 |
| **推荐理由 (Expression)** | 朋友式的推荐文案，解释为什么觉得你会喜欢 |
| **主题标签 (Topic Label)** | 这条内容属于什么主题方向 |
| **互动按钮** | 喜欢 / 不感兴趣 / 稍后看 / 收藏 / 聊一聊 |

### 4.2 推荐理由是怎么生成的

推荐理由的生成经过了精心设计，确保每一条都是**只属于你的个性化表达**：

#### Step 1: 画像注入
系统将你的灵魂画像摘要注入推荐引擎，包含：
- 你的核心兴趣标签（按权重排序，最多 256 个）
- 你不喜欢的话题（最多 128 个，确保推荐文案避开这些）
- 你的 MBTI 人格类型和认知风格
- 你近期的觉察笔记和活跃的洞察假设
- 你的价值观和深层需求

#### Step 2: 内容匹配
对每条候选内容，系统分析：
- 内容主题与你的哪些兴趣方向相关
- 内容的风格是否符合你的口味偏好
- 这条内容在你最近已看内容中是否有新鲜感
- 这条内容是否能满足你当前的深层需求（比如工作压力大时需要解压类内容）

#### Step 3: LLM 生成朋友式表达
推荐文案遵循以下原则：

> **"我觉得"** — 有主观判断，像朋友在说话
> **"我理解你"** — 展示对用户的深层理解
> **关联洞察** — 不只是"你看过类似的"，而是"我理解你为什么喜欢"
> **个性化** — 每一条推荐都只属于这个用户

#### Step 4: 负反馈避让
推荐文案会主动避开你不喜欢的话题和话术模式。如果候选内容明显命中你的避雷项，文案会转为保守说明差异化理由，而非热情背书。

#### Step 5: 批量文案质量保障
- 批量生成时，每条推荐都带 `bvid/content_id`，按 ID 绑定写回，避免"文案张冠李戴"
- 同一句文案被分配给多个不同视频时，整组丢弃（宁可不发也不发重复）
- 大模型返回乱序、漏项时降级为逐条生成

### 4.3 推荐文案的语气风格

推荐文案的语气不是固定模板，而是根据你的**画像、偏好和近期反馈动态调整**：

| 维度 | 说明 | 示例影响 |
|------|------|---------|
| **信息密度** | 解释多详细 | 分析型用户 → 详细推理；休闲型用户 → 简洁直给 |
| **温度** | 多有人情味 | 喜欢温暖表达的用户 → "我觉得你会爱上这个" |
| **梗感** | 多有趣/有梗 | 二次元用户 → 适当玩梗；严肃型用户 → 减少玩笑 |
| **直给程度** | 多直接 | 偏好明确的老用户 → 直接说重点；新用户 → 温和探索 |

## 5. 跨平台内容聚合

### 5.1 已支持的平台

| 平台 | 内容类型 | 发现方式 | 行为采集 |
|------|---------|---------|---------|
| **B 站** | 视频 | API 搜索 + 排行榜 + 相关链 + 扩展 DOM 兜底搜索 | 浏览器插件采集点击/观看/搜索/收藏/投币 |
| **小红书** | 笔记 | 扩展代理搜索 + 创作者订阅 + 初始化导入 | 插件采集收藏/点赞/浏览历史 |
| **抖音** | 短视频 | 扩展服务搜索 + 热点关联 + 首页 Feed | 插件采集发布/收藏/点赞/关注 |
| **YouTube** | 视频 | 后端 yt_search/trending/channel + Google Takeout 导入 | 插件 DOM 读取观看历史/订阅/点赞 |
| **X (Twitter)** | 推文/Thread | 服务端 cookie 重放搜索 + For-You 流 + 账号订阅 | 扩展 GraphQL 被动监听互动 |

### 5.2 跨平台聚合的价值

- **打破平台孤岛**：你在 B 站的兴趣可以影响小红书和 YouTube 的推荐
- **统一画像**：所有平台的行为数据汇入同一个灵魂画像，形成完整的你
- **统一推荐入口**：桌面端 Web、移动端 Web、浏览器插件三端统一，同一套推荐结果
- **来源多样性保障**：推荐批次会主动均衡不同平台来源，避免单一平台刷屏

### 5.3 默认平台配比

系统按可配置的平台配比补充候选池，默认保存的份额为：
- B 站 : 小红书 : 抖音 : YouTube : X = **5 : 1 : 1 : 1 : 1**

但默认只启用 B 站。关闭的平台不会占用候选池配额。用户可在配置中按需启用/禁用任意平台。

## 7. 功能模块详解

### 7.1 🧠 用户灵魂引擎 (Soul Engine)

**目标**：从"他做了什么"到"他为什么这样"到"他是一个什么样的人"。

| 子模块 | 功能 |
|--------|------|
| **PreferenceAnalyzer** | 从行为事件中提取和合并偏好标签，带权重和时间衰减 |
| **AwarenessAnalyzer** | 基于近期事件生成结构化觉察笔记，发现兴趣趋势变化 |
| **InsightAnalyzer** | 基于觉察、偏好和画像生成洞察假设，解释行为背后的动机 |
| **ProfileBuilder** | 构建五层洋葱画像（核心层→价值观层→兴趣层→角色层→表层） |
| **MBTI 推断** | 从行为模式中推断 MBTI 人格类型和维度强度 (E/I, S/N, T/F, J/P) |
| **InterestSpeculator** | 主动猜测用户可能感兴趣的新方向，心理学桥接推理 |
| **AvoidanceSpeculator** | 主动确认用户可能想避开的内容方向 |
| **SocraticDialogue** | 苏格拉底式对话，通过追问深化理解用户 |
| **ToneProfile** | 从画像、偏好和反馈推断推荐语气的密度/温度/梗感/直给程度 |
| **ProfileOverrides** | 用户手动编辑覆盖层，画像重建时不丢失用户调整 |

### 7.2 🔍 内容发现引擎 (Discovery Engine)

**目标**：像资深用户 + 专业编辑一样，通过多种方式主动发现好内容。

| 策略 | 说明 |
|------|------|
| **SearchStrategy** | 从灵魂画像生成关键词，搜索各平台 |
| **TrendingStrategy** | 扫描热门/排行榜，用画像过滤 |
| **ExploreStrategy** | 跨领域探索，刻意离开舒适区 |
| **RelatedChainStrategy** | 从好内容出发沿相关推荐链深入 |
| **XHS/Douyin/YouTube/X Producers** | 各平台专用发现生产者，按缺口独立补池 |
| **统一待评估池** | 不同来源的 raw candidates 统一入池，由 LLM 混合 batch 评估 |

### 7.3 📬 推荐引擎 (Recommendation Engine)

**目标**：从候选池中选出最值得推的内容，并生成朋友式推荐表达。

| 功能 | 说明 |
|------|------|
| **PoolCurator 五维评分** | relevance · freshness · topic_fatigue · source_monotony · serendipity |
| **双轴疲劳控制** | 同时在细粒度 topic_key 和粗粒度 topic_group 上计算疲劳度，取最大值 |
| **MMR 多样化** | Maximum Marginal Relevance 算法，确保同一批推荐真正多样 |
| **来源均衡** | 优先补齐不同来源，再限制重复风格 |
| **新兴趣放大保护** | 刚确认的新兴趣有 24h 滚动预算，避免短期刷屏 |
| **批量文案生成** | 异步预生成推荐文案，serve 时只消费预生成结果 |
| **负反馈避让** | 文案避开 dislike 主题，候选命中避雷项时只能保守说明差异 |

### 7.4 🔄 反馈学习系统 (Feedback Loop)

| 反馈类型 | 采集方式 | 影响 |
|---------|---------|------|
| **隐式反馈** | 浏览器插件自动采集点击/观看时长/滚动 | 影响偏好层权重和兴趣衰减 |
| **显式反馈** | 点赞/踩/不感兴趣/多来点/少来点 | 直接影响偏好层和 disliked_topics |
| **对话反馈** | 在聊天中告诉 Agent 你喜欢什么 | 进入对话洞察分析器，可能产生新兴趣信号 |
| **探针确认** | 对系统猜测的兴趣方向点击"是/不是" | 是→转正为正式兴趣；不是→加入冷却列表 |
| **推荐点击** | 点击推荐卡片观看 | 跨平台统一事件写入，保留 source_platform |

### 7.5 🔧 Skill 系统

支持自定义扩展能力。每个 Skill 是独立模块，包含说明文档 + 执行逻辑。

- **内置 Skill**：B 站搜索、B 站浏览、评论区分析、UP 主追踪等
- **自定义 Skill**：用户和社区可以创建新 Skill 扩展 Agent 能力
- **集成边界**：通过 `integrations/` 模块将 runtime/engine 能力暴露为 OpenClaw 可调用的 Skill

### 7.6 🖥️ 三端交互

| 端 | 入口 | 功能 |
|----|------|------|
| **浏览器插件** | Chrome Extension (MV3) | 跨平台行为采集 + 推荐展示 + 对话/反馈 + Cookie 同步 + 初始化导入 |
| **桌面端 Web** | `http://127.0.0.1:8420/web` | 推荐首页 Hero + 网格 + 画像侧栏 + Runtime 看板 |
| **移动端 Web** | `http://<IP>:8420/m/` | 手机推荐页 + 画像页 + 对话页 |

### 7.7 🔒 隐私与安全

| 特性 | 说明 |
|------|------|
| **100% 本地存储** | 所有数据在本地 SQLite 文件中 |
| **API Key 自主管理** | LLM 默认用你自己的 API Key |
| **API Auth Gateway** | 可选的密码门禁，本机/扩展免登录，LAN/远程需密码 |
| **图片代理** | 封面图通过后端白名单 CDN 代理，保护隐私 |
| **无云端依赖** | 不依赖任何云服务，完全本地运行 |

## 附录：设计原则

1. **灵魂优于标签** — 理解一个人，而不是给他贴标签
2. **有温度的表达** — Agent 的每一次输出都像朋友在说话
3. **主动追问和假设** — 不等用户说，主动猜测并验证
4. **用户掌控权** — 用户可以查看、修正、引导 Agent 的理解
5. **隐私本地化** — 所有数据和计算在本地
6. **开放可扩展** — 通用开源设计 + Skill 系统
